# 02 · Grille d'évaluation du jury + auto-notation MBOLO

> Le formulaire TEFConnect est noté par des évaluateurs sur une série de critères.
> Voici la grille type, ce que le jury cherche concrètement, et comment MBOLO se
> positionne (note indicative /5 et comment maximiser).

## Grille d'évaluation

### 1. Faisabilité / viabilité de l'idée (le « ça tient debout ? »)
**Ce que le jury cherche :** un problème réel, une solution claire, exécutable avec
les ressources prévues.
**MBOLO : 5/5.** Problème concret (accès aux soins/médicaments au Gabon), solution
opérationnelle **déjà développée**. → *Joindre des captures de l'app comme preuve.*

### 2. Opportunité de marché / taille
**Ce que le jury cherche :** un marché large et accessible, une demande démontrée.
**MBOLO : 4/5.** Marché santé + e-logistique gabonais et CEMAC. → *Chiffrer le marché
(population urbaine, taux de pénétration mobile money, nb de pharmacies).*

### 3. Scalabilité / potentiel de croissance
**Ce que le jury cherche :** un modèle réplicable (autres villes / pays) sans tout
reconstruire.
**MBOLO : 5/5.** Marketplace logicielle = coût marginal faible ; réplicable
ville par ville, puis CEMAC. → *Insister sur « 1 APK multi-rôles + 1 API ».*

### 4. Modèle économique & compréhension financière
**Ce que le jury cherche :** comment tu gagnes de l'argent, unit economics, chemin
vers la rentabilité.
**MBOLO : 4/5.** Commissions + frais de service/livraison, **zéro marge sur les
médicaments** (conformité). → *Voir [`05-modele-economique-finances.md`](05-modele-economique-finances.md).*

### 5. Potentiel de leadership / entrepreneur
**Ce que le jury cherche :** la capacité du fondateur à exécuter, résilience, vision.
**MBOLO : à toi de jouer.** → *Raconte ton parcours, pourquoi toi, ce que tu as déjà
livré (l'app construite est une preuve d'exécution rare).*

### 6. Innovation
**Ce que le jury cherche :** une approche nouvelle/maligne pour le contexte local.
**MBOLO : 5/5.** Téléconsultation + ordonnance numérique + livraison + transport
médical + repas, **intégrés**, paiement mobile money (Airtel/Moov), le tout
conforme au rôle légal du pharmacien.

### 7. Impact social & création d'emplois
**Ce que le jury cherche :** emplois créés, bénéfice pour la communauté.
**MBOLO : 5/5.** Coursiers rémunérés, pharmacies/médecins/cuisines partenaires,
meilleur accès aux soins. → *Chiffrer : « X coursiers en an 1, Y partenaires ».*

## Synthèse de positionnement

| Critère | Note indicative |
|---------|-----------------|
| Faisabilité | ★★★★★ |
| Marché | ★★★★☆ |
| Scalabilité | ★★★★★ |
| Modèle économique | ★★★★☆ |
| Leadership | (dépend du pitch fondateur) |
| Innovation | ★★★★★ |
| Impact / emplois | ★★★★★ |

## Les 3 leviers qui feront la différence

1. **Montrer la traction / l'exécution** : l'app est construite → captures d'écran,
   démo, premiers utilisateurs/partenaires signés (même lettres d'intention).
2. **Chiffrer le marché et les finances** : des nombres crédibles rassurent le jury.
3. **Storytelling du fondateur** : pourquoi toi, pourquoi maintenant, ta résilience.

## Erreurs qui font perdre des points (à éviter)

- Réponses vagues / génériques (« je veux aider les gens »). → Sois **spécifique et chiffré**.
- Modèle de revenus flou. → Explique clairement chaque source de revenus.
- Prétendre vendre des médicaments avec marge. → **Non** : reste sur le rôle de
  plateforme logistique, le pharmacien dispense. C'est un **atout de conformité**.
- Copier-coller un business plan générique. → Personnalise au contexte gabonais.
